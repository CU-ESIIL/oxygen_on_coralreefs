# How This Project Works

- Communication: Add the project Slack channel, email thread, or meeting space when ready.
- Documents: Keep public-facing documentation in `docs/`; keep large or sensitive working materials in the appropriate shared storage location.
- Meetings: Store meeting summaries in `docs/resources/` or `templates/meeting-notes/` so decisions remain easy to find.
- Collaboration plan: Use pull requests or focused commits for changes that affect analysis, data processing, or the public website.
- Contributor expectations: Document data provenance, avoid committing large derived files unless they are intentionally part of the public record, and keep the website aligned with the repository.
